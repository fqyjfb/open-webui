# 为 Open WebUI 贡献

🚀 **欢迎，贡献者！** 🚀

非常感谢您对为 Open WebUI 做出贡献的兴趣。本文档旨在指导您完成整个流程，确保您的贡献能够有效地增强项目。让我们一起让 Open WebUI 变得更好！

## 📌 关键点

### 🦙 Ollama vs. Open WebUI

区分 Ollama 和 Open WebUI 至关重要：

- **Open WebUI** 专注于提供直观、响应迅速的网页界面用于聊天交互。
- **Ollama** 是驱动这些交互的底层技术。

如果您的问题或贡献直接与核心 Ollama 技术有关，请将其提交到相应的 [Ollama 项目仓库](https://ollama.com/)。Open WebUI 的仓库仅专注于网页界面方面。

### 🚨 报告问题

发现了问题？有想法？请查看我们的 [Issues 标签](https://github.com/open-webui/open-webui/issues)，看看它是否已经被报告或建议过。如果没有，请随时打开一个新问题。报告问题时，请遵循我们的问题模板。这些模板旨在确保从一开始就提供所有必要的详细信息，使我们能够更高效地解决您的问题。

> [!IMPORTANT]
>
> - **模板合规性**：请注意，如果不遵循提供的问题模板，或根本不提供请求的信息，您的问题可能会在没有进一步考虑的情况下被关闭。这种方法对于维护问题跟踪的可管理性和完整性至关重要。
> - **细节是关键**：为了确保您的问题被理解并能得到有效解决，必须包含全面的细节。描述应清晰，包括重现步骤、预期结果和实际结果。缺乏足够的细节可能会阻碍我们解决您的问题。

> [!WARNING]
> 不希望通过 Issues 报告漏洞！
> 相反，请[使用安全报告功能](https://github.com/open-webui/open-webui/security)，并确保您遵守概述的要求。

### 🧭 支持范围

我们注意到，与 Open WebUI 本身无关，而是与其运行环境相关的问题有所增加，尤其是 Docker 设置。虽然我们努力支持 Docker 部署，但了解 Docker 基础知识对于流畅体验至关重要。

- **Docker 部署支持**：Open WebUI 支持 Docker 部署。我们假设您熟悉 Docker。有关 Docker 基础知识，请参考 [官方 Docker 文档](https://docs.docker.com/get-started/overview/)。

- **高级配置**：为 HTTPS 设置反向代理和管理 Docker 部署需要基础知识。有许多在线资源可用于学习这些技能。确保您具备这些知识将极大地增强您使用 Open WebUI 和类似项目的体验。

- **查看文档并帮助改进它**：[我们的文档](https://docs.openwebui.com) 有不断增长的故障排除指南和详细的安装教程。请验证它是否对您的问题有所帮助，并通过在我们的 [文档仓库](https://github.com/open-webui/docs) 上提交问题和 PR 来帮助扩展它。

### 🌐 翻译和国际化

帮助我们让 Open WebUI 面向更广泛的受众。在本节中，我们将指导您完成向项目添加新翻译的过程。

我们使用 JSON 文件存储翻译。您可以在 `src/lib/i18n/locales` 目录中找到现有的翻译文件。每个目录对应一种特定语言，例如，`en-US` 代表英语（美国），`fr-FR` 代表法语（法国）等。您可以参考 [ISO 639 语言代码](http://www.lingoes.net/en/translator/langcode.htm) 查找特定语言的适当代码。

添加新语言：

- 在 `src/lib/i18n/locales` 路径中创建一个新目录，以适当的语言代码作为其名称。例如，如果您要添加西班牙语（西班牙）的翻译，请创建一个名为 `es-ES` 的新目录。
- 将美式英语翻译文件（来自 `src/lib/i18n/locale` 中的 `en-US` 目录）复制到这个新目录，并根据您的语言以 JSON 格式更新字符串值。确保保留 JSON 对象的结构。
- 将语言代码及其相应的标题添加到 `src/lib/i18n/locales/languages.json` 文件中。

> [!NOTE]
> 添加新翻译时，请在独立的 PR 中进行！功能 PR 或修复错误的 PR 不应包含翻译更新。始终保持 PR 的范围狭窄。
